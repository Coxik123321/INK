# ТЕХНИЧЕСКОЕ ЗАДАНИЕ
на создание ИАСППР «Pipeline AI DSS»

## 1. Общие сведения
Наименование системы: Интеллектуальная система поддержки принятия решений
в области обеспечения целостности трубопроводов (Pipeline AI DSS).

Основание для разработки:
- требования промышленной безопасности
- СП 284.1325800.2016
- ASME B31G
- API 579
- ISO 31000

Разработчик: __________________  
Заказчик: _____________________

## 2. Назначение и цели системы
Система предназначена для:
- оценки технического состояния трубопроводов;
- прогнозирования роста дефектов;
- расчета остаточного ресурса;
- поддержки принятия решений с учетом экономических факторов;
- визуализации результатов в ГИС.

Цель создания системы:
- снижение аварийности;
- оптимизация затрат на ремонт;
- повышение обоснованности управленческих решений.

## 3. Характеристика объекта автоматизации
Объект автоматизации — магистральные и промысловые трубопроводы
нефтяной и газовой промышленности.

## 4. Требования к системе

### 4.1 Функциональные требования
Система должна обеспечивать:
- расчет прочности по B31G, Modified B31G, API 579;
- нечеткую оценку риска на основе НТД;
- прогноз роста дефектов;
- расчет экономических показателей;
- формирование рекомендаций;
- экспорт данных в ArcGIS Pro 3.3.

### 4.2 Требования к надежности
- сохранность данных при сбоях;
- резервное копирование БД.

### 4.3 Условия эксплуатации
- локальный сервер предприятия;
- отсутствие облачных сервисов;
- работа в защищенном контуре.

## 5. Состав и содержание работ
1. Анализ НТД  
2. Формализация требований  
3. Разработка модулей  
4. Тестирование  
5. Ввод в эксплуатацию  

## 6. Порядок контроля и приемки
Приемка осуществляется по результатам:
- функциональных испытаний;
- контрольных расчетов;
- анализа отчетной документации.
